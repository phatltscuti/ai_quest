Claude Code Agent Teams: Executive Summary và Hướng dẫn

1. Executive Summary

Agent Teams là tính năng thử nghiệm của Claude Code cho phép điều phối nhiều phiên Claude Code hoạt động song song như một đội. Một phiên đóng vai team lead, phân công công việc và tổng hợp kết quả; các teammate là các phiên Claude Code độc lập, mỗi phiên có context window riêng, có thể gửi tin nhắn trực tiếp cho nhau và tự phối hợp qua shared task list. Khác với Subagents (chạy trong cùng session và chỉ báo cáo về main agent), Agent Teams cho phép bạn tương tác trực tiếp với từng teammate mà không cần qua lead.

Agent Teams phù hợp nhất khi parallel exploration mang lại giá trị thực: research/review đa góc nhìn, phát triển feature mới (mỗi teammate phụ trách một phần), debug với nhiều giả thuyết song song, hay thay đổi xuyên frontend/backend/tests. Tính năng đang tắt mặc định và có một số hạn chế về session resumption, task coordination, shutdown.

2. Tính năng chính và điểm tiện lợi

2.1. Kiến trúc

  • Team lead: Phiên chính tạo team, spawn teammates, điều phối công việc.
  • Teammates: Các phiên Claude Code độc lập, mỗi phiên làm việc trên task được giao.
  • Task list: Danh sách task dùng chung, teammates tự claim hoặc lead gán.
  • Mailbox: Hệ thống tin nhắn giữa các agent.

2.2. Điểm tiện lợi

  • Giao tiếp peer-to-peer: Teammates nhắn tin trực tiếp cho nhau, không chỉ qua lead.
  • Self-coordination: Shared task list, dependencies tự động unblock khi task phụ thuộc hoàn thành.
  • Tương tác trực tiếp: Bạn có thể chọn teammate và gửi lệnh riêng (Shift+Up/Down trong in-process mode).
  • Plan approval: Có thể yêu cầu teammate lên plan trước, lead duyệt rồi mới implement.
  • Delegate mode: Lead chỉ điều phối, không tự implement, tránh “lead tự làm thay teammate”.
  • Display modes: In-process (mọi terminal) hoặc split panes (tmux / iTerm2) để xem output nhiều teammate cùng lúc.
  • Hooks: TeammateIdle, TaskCompleted để enforce quality gate (feedback, chặn hoàn thành nếu chưa đạt).

3. So sánh Agent Teams vs Subagents

Khía cạnhSubagentsAgent Teams
ContextContext window riêng; kết quả trả về callerContext window riêng; hoàn toàn độc lập
Giao tiếpChỉ báo cáo kết quả về main agentTeammates nhắn tin trực tiếp cho nhau
Điều phốiMain agent quản lý toàn bộShared task list, self-coordination
Phù hợp khiTask tập trung, chỉ cần kết quảCông việc phức tạp cần thảo luận và hợp tác
Chi phí tokenThấp hơn: kết quả tóm tắt về main contextCao hơn: mỗi teammate là một phiên Claude riêng

Dùng Subagents khi cần worker nhanh, tập trung: research câu hỏi, verify claim, review file. Subagent làm xong và trả về tóm tắt.

Dùng Agent Teams khi teammates cần chia sẻ phát hiện, thách thức lẫn nhau, và tự phối hợp. Ví dụ: research với competing hypotheses, parallel code review (security / performance / test coverage), phát triển feature mới (mỗi teammate phụ trách một module).

Chuyển đổi: Nếu chạy nhiều subagents song song và chạm context limit, hoặc subagents cần giao tiếp với nhau, Agent Teams là bước tiếp theo hợp lý.

4. Cài đặt và hoạt động trong môi trường của bạn

4.1. Bật Agent Teams

Agent Teams tắt mặc định. Bật bằng biến môi trường CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1:

  • settings.json (Claude Code): {"env": {"CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"}}
  • Shell: export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 (Linux/macOS) hoặc set CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 (Windows CMD)

4.2. Chạy team đầu tiên

Mô tả task và cấu trúc team bằng natural language, ví dụ:

I'm designing an English center management system (courses, students, teachers). 
Create an agent team to explore this from different angles: one teammate on UX/user flow, 
one on technical architecture, one playing devil's advocate.

Claude tạo team, spawn teammates, điều phối theo prompt. Lead hiển thị danh sách teammates và task; dùng Shift+Up/Down để chọn teammate và nhắn trực tiếp.

4.3. Display mode

  • In-process: Tất cả teammate chạy trong terminal chính. Hoạt động mọi terminal.
  • Split panes: Mỗi teammate một pane. Cần tmux hoặc iTerm2 (it2 CLI). Mặc định "auto": dùng split nếu đang trong tmux, ngược lại in-process.

Để ép in-process: {"teammateMode": "in-process"} trong settings.json hoặc claude --teammate-mode in-process.

4.4. Lưu trữ team & task

  • Team config: ~/.claude/teams/{team-name}/config.json
  • Task list: ~/.claude/tasks/{team-name}/

4.5. Hạn chế (limitations)

  • Không resume in-process teammates khi dùng /resume, /rewind.
  • Task status có thể lag; teammate đôi khi không mark complete, block task phụ thuộc.
  • Một session chỉ quản lý một team; phải cleanup trước khi tạo team mới.
  • Teammates không spawn team hay teammate khác; chỉ lead quản lý.
  • Lead cố định; không promote teammate lên lead.
  • Split panes cần tmux hoặc iTerm2; không hỗ trợ VS Code integrated terminal, Windows Terminal, Ghostty.

5. Ý kiến: Cách dùng trong dự án của chúng ta

5.1. Use case phù hợp

  • Code review song song: 3 reviewer – security, performance, test coverage – mỗi người một góc nhìn, lead tổng hợp.
  • Debug competing hypotheses: Nhiều teammate điều tra giả thuyết khác nhau, tranh luận để loại trừ, tìm root cause nhanh hơn.
  • Feature mới / module mới: Mỗi teammate phụ trách một phần (frontend, backend, test) để tránh conflict file.
  • Refactor lớn: Chia module refactor thành task độc lập, mỗi teammate một tập file riêng.

5.2. Khuyến nghị

  • Bắt đầu với research/review: Nếu mới dùng Agent Teams, nên bắt đầu với PR review, research library, hay investigate bug thay vì implement code ngay. Các task này ranh giới rõ, ít conflict file.
  • Tránh chỉnh cùng file: Phân công sao cho mỗi teammate sở hữu một tập file riêng để tránh overwrite.
  • Plan approval cho task rủi ro: Với refactor lớn hoặc thay đổi auth/DB, yêu cầu teammate nộp plan trước, lead duyệt rồi mới implement.
  • Delegate mode khi cần: Nếu lead hay tự implement thay vì chờ teammate, bật delegate mode (Shift+Tab) để ép lead chỉ điều phối.
  • Kiểm soát chi phí: Agent Teams tốn token nhiều hơn single session. Dùng cho task thực sự hưởng lợi từ parallel exploration; task thường nhật nên dùng single session hoặc Subagents.

6. Demo

Phần demo minh họa luồng: bật Agent Teams → tạo team → xem task list → chọn teammate → nhắn tin trực tiếp → shutdown request từ team-lead.

Hình 1: Cấu hình bật Agent Teams

Thêm CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 vào settings.json hoặc biến môi trường.

Hình 2: Khởi động Claude Code

Chạy Claude Code trong thư mục project, sẵn sàng nhận prompt.

Hình 3: Prompt tạo agent team

Gửi prompt mô tả task và cấu trúc team (ví dụ: 3 teammates – UX, architecture, devil's advocate).

Hình 4: Lead spawn teammates

Lead tạo team, spawn teammates và hiển thị danh sách.

Hình 5: Task list (Ctrl+T)

Shared task list điều phối công việc giữa teammates.

Hình 6: Chọn teammate (Shift+Up/Down)

Chọn teammate để xem và tương tác trực tiếp trong in-process mode.

Hình 7: Nhắn tin trực tiếp cho teammate

Gửi message trực tiếp cho teammate mà không qua lead.

Hình 8: Shutdown request từ team-lead

Khi tất cả tasks hoàn thành, lead tự gửi shutdown request (ví dụ: "Shutdown request from team-lead – Reason: All tasks complete..."). Teammates nhận thông báo và dừng hoạt động.

7. Kết luận

Agent Teams là tính năng mạnh mẽ cho phép điều phối nhiều phiên Claude Code song song, với team lead, shared task list, và giao tiếp peer-to-peer giữa teammates. Điểm mạnh: phù hợp cho parallel research (nhiều góc nhìn cùng lúc), code review đa tiêu chí (security, performance, test coverage), debug với competing hypotheses, và phát triển feature mới (mỗi teammate một module riêng).

So với Subagents: dùng Subagents khi chỉ cần worker nhanh, tập trung, trả về tóm tắt; dùng Agent Teams khi teammates cần chia sẻ phát hiện, thách thức lẫn nhau và tự phối hợp. Chi phí token cao hơn nhưng bù lại giá trị cho bài toán phức tạp.

Lưu ý: Agent Teams đang ở chế độ experimental, cần bật CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 và chấp nhận các hạn chế (session resumption, task status lag, split panes chỉ hỗ trợ tmux/iTerm2 trên một số OS). Nên bắt đầu với task research/review, tránh chỉnh cùng file, và giám sát tiến độ để tối ưu hiệu quả.

Demo trên đây minh họa luồng cơ bản từ bật tính năng → tạo team → xem task list → chọn teammate → nhắn tin trực tiếp → shutdown. Bạn có thể áp dụng tương tự cho các project như quản lý trung tâm tiếng Anh, thiết kế kiến trúc mới, hoặc review PR lớn.
