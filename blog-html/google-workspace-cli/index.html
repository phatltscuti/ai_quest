1. Tổng quan: gws là gì và làm được gì?

Google Workspace CLI (gws) là một công cụ dòng lệnh cho phép bạn tương tác trực tiếp với hầu hết mọi dịch vụ Google Workspace ngay từ terminal, bao gồm:

  • Google Drive – liệt kê, tìm kiếm, tải lên, tải xuống, tạo, xóa, phân quyền file/thư mục
  • Gmail – đọc, tìm kiếm, phân loại, gửi, trả lời, forward email
  • Google Calendar – xem lịch, tạo sự kiện, quản lý lời mời
  • Google Docs / Sheets / Slides – tạo, đọc, chỉnh sửa nội dung tài liệu
  • Google Chat – gửi tin nhắn vào Space, tạo Space mới
  • Admin SDK – quản lý user, group, đơn vị tổ chức (dành cho Google Workspace Admin)
  • Google Tasks, Meet, Classroom và nhiều dịch vụ khác

Điểm quan trọng: đây không phải là một wrapper đơn giản bọc quanh API. gws được xây dựng theo kiến trúc hoàn toàn khác biệt, mà tôi sẽ giải thích ở phần tiếp theo.

2. Kiến trúc và điểm khác biệt cốt lõi

2.1. Dynamic Command Generation (Không hardcode API)

Phần lớn các CLI thông thường hardcode danh sách lệnh và tham số vào source code. Khi Google cập nhật API, nhóm maintainer phải cập nhật CLI và release phiên bản mới.

gws làm ngược lại hoàn toàn. Khi bạn chạy một lệnh, nó tự động gọi Google Discovery Service – một API đặc biệt của Google cung cấp toàn bộ metadata về tất cả các API (endpoints, parameters, schemas) theo thời gian thực. gws đọc metadata đó và sinh ra lệnh tương ứng ngay tại runtime.

Điều này có nghĩa là khi Google ra mắt API mới, gws hỗ trợ ngay lập tức mà không cần update. Đây là lý do gws cực kỳ "future-proof".

2.2. Structured Output (JSON, Table, YAML, CSV)

gws trả về output có cấu trúc, mặc định là JSON. Điều này làm cho nó cực kỳ phù hợp để kết hợp với các công cụ khác:

  • Pipe với jq trên Linux/Mac để filter và transform JSON
  • Pipe với ConvertFrom-Json trong PowerShell để xử lý dữ liệu
  • Dùng flag --format table để hiển thị dạng bảng dễ đọc
  • Xuất --format csv để import vào Excel

2.3. AI-Ready với MCP Server

gws tích hợp sẵn Model Context Protocol (MCP) Server, cho phép các AI Agent như Gemini hoặc Claude kết nối trực tiếp vào gws và thao tác với dữ liệu Google Workspace của bạn. LLM có thể đặt câu hỏi kiểu "hãy tìm email từ sếp tôi trong tuần này và tóm tắt" mà không cần code thêm gì.

2.4. Agent Skills – Hơn 100 kỹ năng tắt

Bên cạnh các lệnh API raw, gws còn có hệ thống Skills – các lệnh tắt được thiết kế sẵn cho các tác vụ phổ biến. Ví dụ:

  • gws gmail +triage – Tóm tắt email chưa đọc ra dạng bảng (thay vì trả về JSON raw phức tạp)
  • gws gmail +read – Đọc nội dung một email cụ thể, tự động decode base64 và extract text
  • gws gmail +watch – Lắng nghe email mới đến và stream chúng theo thời gian thực (NDJSON)
  • gws drive +upload – Upload file với metadata tự động, đơn giản hơn nhiều so với raw API

2.5. Dry-Run Mode

Flag --dry-run cho phép bạn xem chính xác HTTP request nào sẽ được gửi đi (URL, method, headers, body) mà không thực sự thực thi. Cực kỳ hữu ích khi học API hoặc debug.

Ví dụ thực tế tôi đã chạy:

npx @googleworkspace/cli drive files list --dry-run

Kết quả trả về:

{
  "body": null,
  "dry_run": true,
  "is_multipart_upload": false,
  "method": "GET",
  "query_params": [],
  "url": "https://www.googleapis.com/drive/v3/files"
}

3. Cài đặt và Xác thực (Gotcha trên Windows)

Bước 1: Cài đặt gws qua NPM

Mở Windows Terminal (PowerShell) và chạy lệnh sau:

npm install -g @googleworkspace/cli

Hoặc dùng npx để không cần cài đặt toàn cục:

npx @googleworkspace/cli --version

Nếu cài thành công, bạn sẽ thấy:

Bước 2: Tạo OAuth Credentials trên Google Cloud Console

Đây là bước nhiều người bỏ qua và gặp lỗi. gws cần một "OAuth App" trong Google Cloud của bạn để xác thực.

  1. Vào console.cloud.google.com
  2. Tạo hoặc chọn một Project
  3. Vào APIs & Services → Library, bật những API bạn muốn dùng (Google Drive API, Gmail API, Calendar API, v.v.)
  4. Vào APIs & Services → Credentials → Create Credentials → Create OAuth client ID
⚠️ Gotcha #0 – Phải cấu hình Consent Screen trước Nếu đây là Project mới, Google sẽ hiển thị cảnh báo: "To create an OAuth client ID, you must first configure your consent screen" và nút Configure consent screen. Bạn BẮT BUỘC phải làm bước này trước.

✅ Cách xử lý:
  1. Nhấn nút Configure consent screen
  2. Chọn External (nếu dùng tài khoản Gmail cá nhân) hoặc Internal (nếu dùng Google Workspace tổ chức)
  3. Điền App name (ví dụ: "GWS CLI"), User support email và Developer contact email
  4. Nhấn Save and Continue qua các bước Scopes và Test users (có thể bỏ qua, nhấn Next)
  5. Quay lại Credentials → Create Credentials → Create OAuth client ID

4. Lab 1 – Tìm kiếm thư mục trên Google Drive

Bối cảnh: Tôi thường xuyên phải tìm các thư mục dự án cũ trên Drive. Thay vì mở drive.google.com, chờ trang load và dùng thanh tìm kiếm, tôi muốn làm điều đó ngay trong terminal.

Lệnh thực thi

Tìm tất cả thư mục có tên chứa từ "Demo":

gws drive files list --params "{\"q\": \"name contains 'Demo' and mimeType = 'application/vnd.google-apps.folder'\", \"pageSize\": 10}" --format table
⚠️ Gotcha #2 – PowerShell JSON Escaping PowerShell nổi tiếng với việc "ăn mất" dấu nháy kép " khi truyền vào file .exe từ bên ngoài. Nếu bạn dùng JSON chuẩn như {"q": "name contains..."}, gws sẽ báo lỗi: Invalid --params JSON: key must be a string

✅ Giải pháp: Bọc toàn bộ JSON trong dấu nháy đơn '...'. Đối với dấu nháy kép bên trong JSON, dùng \". Đối với chuỗi lồng nhau (nested string) trong Google Query Language, dùng \\\".

5. Lab 2 – Kiểm tra email chưa đọc với Gmail +triage

Bối cảnh: Đầu ngày đi làm, tôi muốn biết ngay sếp có giao task mới không, hay có email quan trọng nào cần xử lý gấp, mà không cần mở tab Gmail lên và mất 1–2 phút chờ load rồi scroll tìm kiếm.

gws có một Skill sẵn cho việc này:

Lệnh thực thi

gws gmail +triage --format table --max 5

Giải thích các tham số:

  • +triage – Skill tắt, tự động gọi API Gmail để lấy email chưa đọc và format gọn
  • --format table – Hiển thị dạng bảng thay vì JSON raw
  • --max 5 – Chỉ lấy tối đa 5 email gần nhất

6. Nhận xét cá nhân và ý tưởng tự động hóa

Theo quan điểm của tôi, gws là một game-changer thực sự. Nó loại bỏ hoàn toàn nhu cầu phải viết 50 dòng boilerplate code Python/Node.js và cấu hình Service Account chỉ để thực hiện một API request đơn giản. Tốc độ rất nhanh và output JSON làm cho nó trở thành mảnh ghép hoàn hảo cho scripting và automation.

Những cách tôi sẽ ứng dụng gws vào công việc hàng ngày:

6.1. "Good Morning" Dashboard

Tôi sẽ viết một script PowerShell chạy tự động mỗi khi mở terminal vào buổi sáng. Script này sẽ:

  • Gọi gws calendar events list --params '{"timeMin": "...", "timeMax": "..."}' để lấy danh sách lịch họp hôm nay
  • Gọi gws gmail +triage --max 10 để hiển thị 10 email chưa đọc quan trọng nhất
  • In ra một bảng tổng hợp ngay trong console, không cần mở bất kỳ tab trình duyệt nào

Ước tính tiết kiệm: 5–10 phút mỗi buổi sáng không phải mở trình duyệt, chờ load, đăng nhập lại.

6.2. Tự động sao lưu Code lên Google Drive

Tôi sẽ tạo một Windows Scheduled Task chạy lúc 5:00 PM mỗi ngày:

  1. Zip thư mục project hiện tại
  2. Dùng gws drive files create để upload file zip lên một thư mục backup chỉ định trên Drive
  3. Xóa các bản backup cũ hơn 30 ngày bằng gws drive files delete

Không cần phần mềm backup thứ ba, không cần cấu hình phức tạp, chỉ cần một script ngắn.

6.3. CI/CD Notification Bot

Sau mỗi lần deploy thành công, pipeline tự động:

  1. Tạo document release note mới trong Google Docs với gws docs
  2. Gửi thông báo vào Google Chat Space của team với gws chat spaces messages create
  3. Update trạng thái deployment vào Google Sheet tracking với gws sheets values append

6.4. Audit Quyền File Drive Hàng Tuần

Mỗi sáng thứ Hai, script tự động quét tất cả file đang được share với quyền "anyone with the link" trên Drive của công ty: 

gws drive files list `
  --params '{\"q\": \"visibility = \\\"anyoneWithLink\\\"\"}' `
  --format table > security_audit.txt

Kết quả được gửi tự động qua email cho IT Admin để review. Không cần dashboard phức tạp, không cần phần mềm thứ ba.

7. Những hạn chế của gws

Để đánh giá khách quan, tôi phải nêu ra những điểm còn hạn chế:

7.1. Syntax Khác Nhau Giữa Shell

Như đã thấy trong Lab 1, cú pháp truyền JSON qua --params trên PowerShell khác hoàn toàn so với Bash. Một script chạy tốt trên Linux hoàn toàn có thể báo lỗi trên Windows PowerShell và ngược lại. Đây không phải lỗi của gws, nhưng là thực tế người dùng phải đối mặt khi làm việc đa môi trường.

7.2. Google API Rate Limits

gws giao tiếp trực tiếp với Google Workspace APIs, nghĩa là bị ràng buộc bởi quota của Google. Nếu bạn viết script bulk thao tác hàng nghìn file (download, upload, xóa) trong vòng lặp chặt, bạn sẽ nhận lỗi 429 Too Many Requests. Cần thêm delay hoặc backoff logic vào script.

7.3. Không Phù Hợp với Nội Dung Rich Media

gws là công cụ text-first. Đọc email HTML phức tạp có ảnh, bảng, định dạng trong terminal là trải nghiệm tệ. Xem Google Slides hay ảnh trên Drive qua CLI cũng vậy. Đây không phải tool để thay thế hoàn toàn trình duyệt, mà để supplement cho các tác vụ dữ liệu.

7.4. Initial Setup Còn Phức Tạp Với Non-Developer

Bước tạo OAuth credentials trên Google Cloud Console, enable API, setup Client ID... là quá trình kỹ thuật mà người dùng thông thường sẽ thấy khó. Hơn nữa, màn hình OAuth consent cảnh báo app "unverified" có thể gây lo lắng không cần thiết. Đây không phải tool dành cho người dùng cuối.

8. Kết luận

Sau khi tự tay thực hiện từng bước cài đặt, xác thực và chạy hai ví dụ thực tế trên Windows PowerShell, tôi nhận thấy gws xứng đáng với danh tiếng của nó.

Điểm mạnh nhất của gws không phải là danh sách tính năng dài, mà là triết lý thiết kế của nó: dynamic, structured output và AI-ready. Ba yếu tố này cộng lại biến nó thành một công cụ không bao giờ lỗi thời và luôn có thể kết hợp với bất cứ công cụ nào khác trong workflow của bạn.

Nếu bạn là developer, sysadmin, automation engineer, hay đơn giản là người thích keyboard hơn chuột – hãy thử gws ngay hôm nay. Khi đã vượt qua bước setup ban đầu (và hiểu cách PowerShell handle JSON), nó sẽ tiết kiệm cho bạn hàng giờ mỗi tuần.
