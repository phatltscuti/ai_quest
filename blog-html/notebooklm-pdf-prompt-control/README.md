# NotebookLM PDF Prompt Control – Kiểm soát Slide Deck qua Prompt Engineering

Blog hướng dẫn chi tiết cách dùng NotebookLM tạo slide deck dạng PDF, tập trung vào kỹ thuật prompt engineering để kiểm soát cấu trúc (outline), giọng điệu (tone), và phong cách thiết kế (visual style).

## AI Quest Requirements

| # | Requirement | Deliverable |
|---|-------------|-------------|
| 1 | Source pack ≥ 5 nguồn (PDF/Docs/Web) + giải thích lý do chọn | Mục 2 trong blog |
| 2 | Prompt kiểm soát (a) outline, (b) tone, (c) visual style + iterate ≥ 2 lần | Mục 3–5 trong blog |
| 3 | Output PDF + screenshots: prompt, slide preview, exported PDF | Mục 6 + thư mục `images/` |
| 4 | Rubric đánh giá (structure / clarity / design / factual) 1–5 + đề xuất cải thiện | Mục 7 trong blog |

## Nội dung

1. **NotebookLM Slide Deck là gì?** – Tổng quan tính năng, workflow, ưu/nhược điểm
2. **Source Pack** – 7 nguồn tài liệu (PDF/Docs/Web/YouTube) cho công ty EdTech mẫu + lý do chọn
3. **Iteration 1** – Prompt V1 kiểm soát Outline + Tone + Visual Style (phân tích chi tiết)
4. **Đánh giá V1 & Chiến lược cải thiện** – Phân tích vấn đề, xây dựng V2
5. **Iteration 2** – Prompt V2 refined + so sánh before/after
6. **Xuất PDF & Screenshots** – Quy trình export + checklist ảnh bắt buộc
7. **Rubric đánh giá & Đề xuất cải thiện** – Chấm điểm 4 tiêu chí + 7 đề xuất nâng cao
8. **Kết luận & Best Practices** – Bài học rút ra + prompt engineering tips
